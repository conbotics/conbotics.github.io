    Advantages of Isaac Sim

    • Photorealism & Ray Tracing: real-time raytracing, global illumination, physically based materials -> suitable for training vision-based AI
      • Isaac Sim can accrutely handle transparency/reflections
    • Isaac Sim is GPU accelerated
    • USD assets are highly portable and modular compared to Gazebos classics SDF/URDF
    • Synthetic Data Generation (SDG) -> Isaac Sim includes Replicator: a tool designed to generate massive datasets for training AI Models
      • with automatic domain randomization (varying lighting, textures, positions)
    • PhysX: Allows for accurate simulation of Robot dynamics

    Solution for given Setup

    1. Ubuntu 24
    2. Containerized Ros 1 Noetic

    Solution #1

    Container-Native Approach - Isaac Sim Version 2022.2.1 or 2023.1.1 - These Version Have Ros 1 Bridge enabled (natively) - Simulator speaks ros 1 - no translation nodes

    • Setup:

      • Run Both (Isaac Sim + ROS Noetic) containerized (shared network)
        • Run Isaac Sim in Headless Mode
          • use Omniverse Streaming Client (installed natively on Ubuntu host)

    • Docker command: docker run --name isaac-sim --entrypoint ./runheadless.native.sh --gpus all -e "ACCEPT_EULA=Y" --rm --net=host nvcr.io/nvidia/isaac-sim:2022.2.1 .OR. 2023.1.1

    Solution #2

    Bridge-Translation Approach - Run Isaac Sim 5.x Natively + ROS 1 Bridge Node - Use ROS 2 Bridge - Isaac Sim publishes to ROS 2 - Intermediate Container Runs ros1_bridge (dynamic_bridge) - Converts from ROS 2 -> ROS 1 - Noetic Container

    • This Method adds latency (Is Isaac Sim 5.x really needed??)
