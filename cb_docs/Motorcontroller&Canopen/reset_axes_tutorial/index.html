    How to reset the axes 1 to 3 of the painting arm

    CAN message structure

    A general explanation of the CAN message structure:

    [60][AXIS_ID]#[MSG_TYPE] [INDEX(REVERSED)] [SUBINDEX] [MESSAGE(REVERSED)]

    • 60: dunno
    • AXIS_ID: 1, 2 or 3 (depending on which axis to send to)
    • MSG_TYPE:
    • 40 = request
    • 43 = response

    • 23 = command

    • INDEX(REVERSED in pairs):

    • 0232 for index 3202 -> automatic brakes
    • fe60 for index 60fe -> manual brakes
    • SUBINDEX:
    • 00 for index 3202 (automatic brakes)
    • 01 for index 60fe (manual brakes)
    • MESSAGE (REVERSED in pairs): to send 00 01 00 00 you actually have to send 01 00 00 00

    Example: 602#23 0232 00 00 00 00 00

    → a message to axis 2 sending a command (23) to index 3202 and subindex 00 with the message 00 00 00 00

    CANdump response structure

    can1 582 [8] 43 02 32 00 05 00 00 00

    Resetting procedure

    The following steps are done for axis 2 as an example, but are transferrable to the other axes.

    Step 1: Disengaging the automatic brake

    Step 1a: Reading out the current value for the automatic brake

    Request to motor controller: cansend can1 602#4002320000000000

    → can message: 60 2 # 40 0232 00 00000000 - axis 2 - request 40 - index 3202 (reversed) - subindex 00 - empty message 00 00 00 00

    Response from motor controller: can 582 [8] 43 02 32 00 05 00 00 00 - response message type (58) - axis 2 - message length 8 - response message type (??) (43) - index 3202 (reversed) - subindex 00 - message 05 00 00 00 → 5hex

    Step 1b: Converting HEX value to binary, inverting bit 2, converting back and sending to motor

    5hex = 0101bin

    • inverting bit 2: 0001
    • converting back to hex: 0001bin = 1hex → can message: 01 00 00 00
    • send back: cansend can1 602#2302320001000000

    Step 2: Dis- and re-engaging the manual brake

    Step 2a: Disengaging the manual brake

    Command to motor controller: cansend can1 602#23fe600100000000

    → can message: 60 2 # 23 fe60 01 00 00 00 00 - axis 2 - command message type (23) - index 60fe (reversed) - subindex 01 - message 00 00 00 00

    Now you can make your manual adjustments to the axis.

    Step 2b: Re-engaging the manual brake

    Command to motor controller: cansend can1 602#23fe600101000000

    → can message: 60 2 # 23 fe60 01 01 00 00 00

    Step 3: Re-engaging the automatic brake

    Step 3a: Sending the command

    Command to motor controller: cansend can1 602#2302320005000000

    → can message: 60 2 # 23 0232 00 05 00 00 00 - axis 2 - command message type (23) - index 3202 (reversed) - subindex 00 - message 05 00 00 00 → as read out in step 1a

    Step 3b: Confirming that re-engaging has worked

    Request to motor controller: cansend can1 602#4002320000000000

    → can message: 60 2 # 40 0232 00 00000000 - axis 2 - request 40 - index 3202 (reversed) - subindex 00 - empty message 00 00 00 00

    Response: can 582 [8] 43 02 32 00 05 00 00 00 - response message type (58) - axis 2 - message length 8 - response message type (??) (43) - index 3202 (reversed) - subindex 00 - message 05 00 00 00
